# Claude-to-Codex Derivation Strategy

## Goal
Keep `Claude Code` as the source of truth and generate `Codex` artifacts as derived outputs.

## Source of Truth
- `CLAUDE.md`
- `.claude/agents/*.md`
- `.claude/skills/*/SKILL.md`
- `.claude/hooks/*.sh` for Claude-only runtime enforcement behavior that must be mirrored in Codex docs/verifiers, not emitted as direct Codex hook targets
- `.claude/settings.json`
- `.claude/settings.local.json`
- `.mcp.json`

## Derived Codex Targets
- `AGENTS.md`
- `.codex/config.toml`
- `.codex/agents/*.toml`
- `.agents/skills/*/SKILL.md`

## Operating Rule
- Edit Claude source files directly.
- Do not hand-edit Codex derived files except for bootstrap experiments.
- If a derived file must be patched, port the same change back to the Claude source and regenerate.

## Change Detection
- The active manifest is `.codex-sync/manifest.json`, regenerated by `scripts/generate_codex_derivatives.sh`.
- `.codex-sync/manifest.template.json` is a bootstrap reference for new repository rollouts; it is not enforced and may diverge from the active manifest.
- Treat manifest drift in the active manifest as a failure, not as advisory information.
- Run a sync check before opening Claude or Codex in any migrated repository.
- Hook files affect Codex indirectly through mirrored runtime docs and verifier expectations. They are source-of-truth for behavior, but not one-to-one manifest targets.
- The `rules` block in `manifest.template.json` is a bootstrap catalog for rollout guidance. It is not enforced by itself unless the same rule is emitted into `.codex-sync/manifest.json` or checked explicitly by a verifier.

## 13-Repository Rollout
1. Copy the same `.codex-sync/` folder into each target repository.
2. Fill the manifest with repository-specific mappings.
3. Keep the source categories identical across all repositories.
4. Allow repository-local exceptions only inside the manifest notes field.

## Non-Goals
- No bidirectional merge workflow.
- No assumption that Claude or Codex will infer stale derived files automatically.
- No shared "neutral" control document that both tools must discover indirectly.
